## Bug ID: BUG-002

| Field | Detail |
|-------|--------|
| **Title** | SearchActivity is not exported, cannot be launched from outside the app |
| **Severity** | Low |
| **Priority** | P3 |
| **Status** | New |
| **Reported By** | Tianyu Yao |
| **Date** | 2026-05-27 |

### Environment
- **Device**: MuMu Emulator, Android 12
- **App Version**: 3.26.052700debug
- **Network**: N/A

### Steps to Reproduce
1. Run in terminal: `adb shell am start -n io.legado.app.debug/io.legado.app.ui.book.search.SearchActivity`
2. Observe the error

### Expected Result
SearchActivity launches and displays the search interface.

### Actual Result
Activity fails to start with `SecurityException: Permission Denial`.
The activity is not exported in AndroidManifest.xml and cannot be launched
by external intents or instrumentation tests that use direct intent launch.

### Screenshots / Logs
```
java.lang.SecurityException: Permission Denial: starting Intent {
    flg=0x10000000 cmp=io.legado.app.debug/...SearchActivity
} from null not exported from uid 10041
```

### Notes
Workaround: launch via MainActivity overflow menu → Search.
This affects testing — TC-006 had to be restructured to go through
the menu flow instead of launching SearchActivity directly.
