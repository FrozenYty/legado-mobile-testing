## Bug ID: BUG-003

| Field | Detail |
|-------|--------|
| **Title** | ReadBookActivity redirects to HandleFileActivity for books with local origin |
| **Severity** | Medium |
| **Priority** | P2 |
| **Status** | New |
| **Reported By** | Tianyu Yao |
| **Date** | 2026-05-27 |

### Environment
- **Device**: MuMu Emulator, Android 12
- **App Version**: 3.26.052700debug
- **Network**: N/A

### Steps to Reproduce
1. Insert a Book into the database with `origin = BookType.localTag` (the default)
2. Set `bookUrl` to a value that does not correspond to an actual file on disk
3. Launch ReadBookActivity with this bookUrl via Intent
4. Observe behavior

### Expected Result
ReadBookActivity opens and displays the book metadata or a "file not found" message.

### Actual Result
ReadBookActivity immediately redirects to HandleFileActivity, which tries to
parse the non-existent file. The test scenario loses the ReadBookActivity
instance and cannot verify the reading view. The content area shows "没有书籍"
with page count "1/0 0.0%".

### Screenshots / Logs
```
Activity lifecycle changed event received but ignored because the intent
does not match. startActivityIntent={ReadBookActivity},
activity.getIntent()={HandleFileActivity}
```

### Notes
Workaround: set `origin = "https://example.com"` (a non-local URL) to prevent
the redirect. This affects all tests that need to verify the reading view
(TC-003 through TC-005).
